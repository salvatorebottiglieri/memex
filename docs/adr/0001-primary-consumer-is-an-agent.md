# Primary consumer is an agent, not a human reader

The system is built to be **queried by an agent** that answers in natural language with citations — not read as a hand-curated human wiki. I personally consult mostly the high-level derivations; the browsable Obsidian markdown is a by-product, not the product.

## Considered Options

- **(A) Human-read wiki** (Karpathy-style): curated prose I read and navigate by hand.
- **(B) Agent-queried corpus** (chosen): structure exists to let an agent navigate and cite.

## Consequences

Optimizing for (B) yields (A) for free (the markdown is browsable). The reverse does not: curating prose first dies by manual-editing attrition — the same death the WhatsApp link graveyard suffers today. This is why "rigorous citation" and "facilitate the agent" are treated as the load-bearing requirements.
